# 图谱分析任务

## 当前需求
$requirement

## 已有图谱节点
$nodes

## 已有图谱边
$edges

## 分析要求

请分析当前需求与已有需求图谱的关系，从以下几个维度进行深入分析：

### 1. 功能依赖关系
- 识别当前需求是否依赖其他需求的功能（A需要B先完成）
- 识别是否有其他需求依赖当前需求的功能
- 考虑技术实现层面的前置条件

### 2. 数据依赖关系
- 分析当前需求与其他需求是否共用同一数据源
- 识别数据流向和数据依赖链
- 考虑数据一致性和同步需求

### 3. 业务互补关系
- 识别哪些需求与当前需求一起上线效果更好
- 分析业务场景上的协同效应
- 考虑用户体验的连贯性

### 4. 资源冲突关系
- 识别是否存在资源竞争（如争夺同一团队资源）
- 分析时间线上的冲突可能性
- 考虑技术栈的兼容性

### 5. 核心业务判断
- 评估当前需求在整体业务中的重要性
- 分析其对下游需求的解锁能力
- 判断其是否为核心枢纽节点

## 输出格式

返回JSON格式：
```json
{
  "newEdges": [
    {"source": "REQ-XXX", "target": "REQ-YYY", "type": "dependency", "label": "依赖", "reason": "功能依赖说明"},
    {"source": "REQ-XXX", "target": "REQ-ZZZ", "type": "complement", "label": "互补", "reason": "业务互补说明"},
    {"source": "REQ-XXX", "target": "REQ-WWW", "type": "conflict", "label": "冲突", "reason": "资源冲突说明"}
  ],
  "isCore": true/false,
  "graphBonus": 8,
  "analysis": "详细的图谱分析说明，包括识别的关键关系和建议"
}
```

## 关系类型说明
- **dependency**: 依赖关系（A依赖B，B必须先完成）
- **complement**: 互补关系（A和B结合效果更好）
- **conflict**: 冲突关系（A和B存在资源或逻辑冲突）
- **weak**: 弱关联（间接相关）
- **sameGoal**: 同目标（服务于相同业务目标）

## 评分标准
- **graphBonus**: 0-20分，根据关系的数量和重要性评分
  - 高连接度（>=3个关系）且为核心节点：15-20分
  - 中等连接度（1-2个关系）：8-14分
  - 低连接度（无直接关系）：0-7分
